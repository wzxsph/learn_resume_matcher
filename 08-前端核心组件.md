# Resume Matcher 前端核心组件深度解析

> 适用版本：`apps/frontend`（Next.js 16 + React 19 + TypeScript + Tailwind v4）
> 本教程面向具备 React 基础、希望深入理解真实业务组件设计的中级开发者。

## 一、组件目录总览

`apps/frontend/components/` 按职责切分为 10 个目录，共约 70 个 `.tsx` 文件。组织原则为"按业务领域 + 按抽象层级"两维划分。

| 目录 | 文件数 | 抽象层级 | 业务领域 |
|------|--------|----------|----------|
| `ui/` | 14 | 原子组件 | 跨页面通用（Button、Dialog、Input） |
| `common/` | 2 | 跨领域工具 | 错误边界、跨页面 Context |
| `home/` | 2 | 落地页 | Hero 横幅、SwissGrid 网格 |
| `dashboard/` | 2 | 仪表盘 | 上传对话框、Resume 类型定义 |
| `builder/` | 21 + 9 forms | 编辑器 | 主简历表单、模板、AI 重生成、JD 对比 |
| `enrichment/` | 4 | AI 增强向导 | 多步问答、AI 改进预览 |
| `tailor/` | 1 | 差异化定制 | 简历前后对比弹窗 |
| `resume/` | 6 + 5 CSS | 渲染模板 | 4 种排版样式的输出组件 |
| `preview/` | 4 | 分页预览 | WYSIWYG 页面分页、缩放 |
| `settings/` | 1 | 设置 | LLM API Key 菜单 |

### 1.1 用户故事视角下的组件映射

整个应用的核心闭环：**上传 → 改进 → 定制 → 导出**。每一阶段由一组组件协作完成。

| 阶段 | 用户动作 | 关键组件 | 入口文件 |
|------|----------|----------|----------|
| 上传 | 拖入 PDF/DOCX | `ResumeUploadDialog` | `components/dashboard/resume-upload-dialog.tsx:40` |
| 改进 | AI 增强简历 | `EnrichmentModal` + `QuestionStep` | `components/enrichment/enrichment-modal.tsx:25` |
| 编辑 | 表单修改 | `ResumeForm` + `SectionHeader` | `components/builder/resume-form.tsx:50` |
| 定制 | 粘贴 JD 生成 | `TailorPage` + `DiffPreviewModal` | `app/(default)/tailor/page.tsx:24` |
| 导出 | 下载 PDF | `PaginatedPreview` + `PageContainer` | `components/preview/paginated-preview.tsx:26` |

## 二、UI 基础组件（`components/ui/`）

UI 组件库采用"手写而非依赖 shadcn" 的策略，每一组件都按 Swiss International Style 严格设计：`rounded-none` 方形硬边、1px 黑边框、`shadow-sw-sm` 硬阴影（无模糊）、hover 时 `translate-[1px]` 制造"按下"反馈。

### 2.1 Button：`components/ui/button.tsx`

定义 8 个语义化变体（`variant`），区分主次操作：

| 变体 | 颜色 | 用途 |
|------|------|------|
| `default` | Hyper Blue `#1D4ED8` | 保存、提交、创建 |
| `success` | Signal Green `#15803D` | 下载、确认、完成 |
| `destructive` | Alert Red `#DC2626` | 删除、移除 |
| `warning` | Alert Orange `#F97316` | 重置、清空 |
| `outline` | 透明 + 黑边框 | 取消、返回 |
| `ghost` | 无背景 | 工具栏图标 |
| `link` | 文字下划线 | 内联链接 |

关键设计：icon 按钮使用 `before:-inset-1.5` 伪元素扩展点击区至 44×44（WCAG 2.5.8 要求），即使可见尺寸只有 32×32。源码注释 `components/ui/button.tsx:67-75` 解释了此设计动机。

```tsx
// components/ui/button.tsx:46
const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant = 'default', size = 'default', ...props }, ref) => {
    // baseStyles: layout, transition, focus, disabled
    // variantClass: bg/border/shadow/hover/active
    // sizeClass: h-X w-X px-X
    return <button ref={ref} className={cn(baseStyles, variantClass, sizeClass, className)} {...props} />;
  }
);
```

### 2.2 Dialog：`components/ui/dialog.tsx`

无外部依赖的原生实现。使用 `React.createContext` 暴露 `open`、`onOpenChange`、`titleId` 三个字段。`DialogContent` 通过 `createPortal` 渲染到 `document.body` 并设置 `role="dialog"` + `aria-modal="true"` + `aria-labelledby` 三件套（`components/ui/dialog.tsx:144-156`）。`useId()` 保证多实例下 `aria-labelledby` 引用稳定（`components/ui/dialog.tsx:43`）。

支持的子组件：`DialogTrigger`、`DialogClose`、`DialogContent`、`DialogHeader`、`DialogFooter`、`DialogTitle`、`DialogDescription`。`DialogTrigger` 支持 `asChild` 模式通过 `React.cloneElement` 注入 `onClick`（`components/ui/dialog.tsx:56-71`），可与任意按钮组合。

### 2.3 ConfirmDialog：`components/ui/confirm-dialog.tsx`

基于 `Dialog` 二次封装，提供 4 种语义（`danger` / `warning` / `success` / `default`），每种对应不同图标和确认按钮变体（`components/ui/confirm-dialog.tsx:73-106`）。`closeOnConfirm` 默认 `true`，意味着点击确认后自动关闭弹窗。

### 2.4 RetroTabs：`components/ui/retro-tabs.tsx`

瑞士风格 Tabs：底部黑边、活动 Tab 白色背景 + 顶左硬阴影模拟凸起（`components/ui/retro-tabs.tsx:50-53`）。`disabled` 状态时背景变为 `bg-paper-tint` + 50% 透明度，鼠标光标变为 `not-allowed`。

| 组件 | 文件 | 关键特性 |
|------|------|----------|

## 三、通用组件（`components/common/`）

### 3.1 ResumePreviewProvider：`components/common/resume_previewer_context.tsx`

跨页面共享 AI 改进结果的 Context。提供 `improvedData` 与 `setImprovedData` 两个字段（`components/common/resume_previewer_context.tsx:107-110`）。`ImprovedResult.data` 包含 LLM 输出的所有字段：`resume_preview`、`cover_letter`、`outreach_message`、`diff_summary`、`detailed_changes`、`job_description`、`job_keywords`（`components/common/resume_previewer_context.tsx:73-101`）。在 `app/(default)/layout.tsx` 包裹整个 `(default)` 路由组，使 `tailor` 页写入的 AI 结果能被 `builder` 页直接读取。

### 3.2 ErrorBoundary：`components/common/error-boundary.tsx`

React 类组件，捕获子树异常并展示回退 UI。`getDerivedStateFromError` 切换 `hasError` 标志（`components/common/error-boundary.tsx:37`），`componentDidCatch` 打印到控制台（`components/common/error-boundary.tsx:41`）。暴露两个动作按钮：Try Again（重置 state）与 Reload Page（`window.location.reload()`）。

## 四、仪表盘（`components/dashboard/`）

### 4.1 ResumeUploadDialog：`components/dashboard/resume-upload-dialog.tsx`

支持受控/非受控两种 open 模式（`components/dashboard/resume-upload-dialog.tsx:54-61`）。内部使用 `useFileUpload` hook 管理文件状态（拖拽、验证、上传进度）。允许的 MIME：`application/pdf`、`application/vnd.openxmlformats-officedocument.wordprocessingml.document`、`application/msword`，单文件 4MB 上限（`components/dashboard/resume-upload-dialog.tsx:33-38`）。

成功上传后延迟 1500ms 关闭弹窗以便用户看到成功提示（`components/dashboard/resume-upload-dialog.tsx:83-90`）。后端返回 `processing_status: 'failed'` 时保留弹窗并暴露"重试处理"按钮（`components/dashboard/resume-upload-dialog.tsx:120-128`）。

### 4.2 Resume 类型与默认组件：`components/dashboard/resume-component.tsx`

文件导出所有共享类型：`PersonalInfo`、`Experience`、`Education`、`Project`、`AdditionalInfo`、`SectionMeta`、`CustomSection`、`ResumeData`。`SectionMeta` 是动态章节系统的核心，记录每个章节的 `id`/`key`/`displayName`/`sectionType`/`isDefault`/`isVisible`/`order`（`components/dashboard/resume-component.tsx:88-96`）。

默认导出的 `Resume` 是一个轻量路由组件，根据 `settings.template` 选择 `swiss-single` / `swiss-two-column` / `modern` / `modern-two-column` 四种模板（`components/dashboard/resume-component.tsx:179-207`）。所有模板接收相同的 `data: ResumeData`，通过 `settingsToCssVars` 将设置转换为 CSS 变量（`components/dashboard/resume-component.tsx:172`），实现"一套数据 + 一组变量 → 四种样式"。

## 五、简历构建器（`components/builder/`）

构建器是整个应用最复杂的模块，21 个文件 + 9 个 form 子组件。`resume-builder.tsx` 是总装入口。

### 5.1 ResumeBuilder：`components/builder/resume-builder.tsx`

主编辑器，1028 行。状态机包含 16 个 `useState`，管理 4 个 Tab（resume / cover-letter / outreach / jd-match）。**数据加载三优先级**逻辑（`components/builder/resume-builder.tsx:282-361`）：

1. URL 中 `?id=` 存在 → 调用 `fetchResume(id)` 拉取后端
2. Context 中的 `improvedPreview`（来自 tailor 页）→ 用作草稿
3. `localStorage['resume_builder_draft']` → 浏览器刷新恢复

**保存/下载逻辑**：`handleSave` 调用 `updateResume`（`components/builder/resume-builder.tsx:411`），`handleDownload` 调用 `downloadResumePdf` 拿到 Blob 再用 `downloadBlobAsFile` 触发浏览器下载（`components/builder/resume-builder.tsx:443-449`）。`onbeforeunload` 事件防止误关丢失草稿（`components/builder/resume-builder.tsx:270-279`）。

**AI 重新生成**使用 `useRegenerateWizard` hook（`components/builder/resume-builder.tsx:160-204`），把 `resumeData.workExperience` / `personalProjects` / `additional.technicalSkills` 映射为 `RegenerateItemInput[]`（`components/builder/resume-builder.tsx:207-238`）。

### 5.2 ResumeForm：`components/builder/resume-form.tsx`

章节表单编排器。使用 `@dnd-kit` 实现拖拽排序，外部 `DndContext` + 内部 `SortableContext` 包裹所有章节（`components/builder/resume-form.tsx:379-405`）。`handleDragEnd` 通过交换 `order` 字段实现重排（`components/builder/resume-form.tsx:173-207`），保留 `personalInfo` 始终在最前。

`renderDefaultSection` 根据 `section.key` 分发到 6 个子表单组件（`components/builder/resume-form.tsx:215-273`）。`renderCustomSection` 根据 `section.sectionType`（`text` / `itemList` / `stringList`）渲染对应的通用表单（`components/builder/resume-form.tsx:318-358`）。所有默认/自定义章节都包裹在 `SectionHeader` 中，提供重命名/上下移/删除/显隐 4 个动作。

### 5.3 SectionHeader：`components/builder/section-header.tsx`

章节元数据操作面板。本地状态 `isEditing` / `editedName` / `showDeleteConfirm` 三个（`components/builder/section-header.tsx:46-48`）。重命名支持 Enter 保存、Esc 取消（`components/builder/section-header.tsx:67-73`）。删除内置二次确认（`ConfirmDialog`），默认章节的删除按钮实际触发"隐藏"动作（`components/builder/section-header.tsx:75-83`）。

### 5.4 DraggableSectionWrapper & DraggableListItem

两个拖拽包装器。前者包装章节（`id: string`），后者包装列表项（`id: number`，如一条工作经历）。两者都用 `@dnd-kit/sortable` 的 `useSortable` hook 获取 `attributes`/`listeners`/`setNodeRef`/`transform`/`transition`/`isDragging`，并将 `transform` 通过 `CSS.Transform.toString` 注入 style（`components/builder/draggable-section-wrapper.tsx:29-39`）。差异在于禁用条件：章节中 `personalInfo` 不可拖动（`resume-form.tsx:395`），列表项均可拖动。

### 5.5 章节表单：`components/builder/forms/`

9 个表单组件，每个都是受控组件。共同模式：父组件持有 `data`，子组件通过 `onChange(data)` 整体回传，避免深拷贝子树。

| 表单 | 文件 | 特殊能力 |
|------|------|----------|
| `PersonalInfoForm` | `forms/personal-info-form.tsx:14` | 8 个字段的网格布局 |
| `SummaryForm` | `forms/summary-form.tsx:13` | 纯 Textarea，Enter 阻止冒泡 |
| `ExperienceForm` | `forms/experience-form.tsx:45` | **动态加载 Tiptap**（`dynamic` import 减少首屏 JS），`DraggableListItem` 排序 |
| `EducationForm` | `forms/education-form.tsx:33` | 同 ExperienceForm 但更简单 |
| `ProjectsForm` | `forms/projects-form.tsx` | 类似 Experience |
| `AdditionalForm` | `forms/additional-form.tsx:14` | 4 个 Textarea 数组字段（每行一项） |
| `GenericTextForm` / `GenericItemForm` / `GenericListForm` | 同目录 | 自定义章节三种 schema 类型的渲染器 |

### 5.6 FormattingControls：`components/builder/formatting-controls.tsx`

模板/排版设置面板（576 行）。可折叠（`isExpanded` state），内含 6 大区块：模板选择、强调色（仅 modern 系列）、页面尺寸、边距、间距、字号。每个区块都遵循相同模式：`<h4>` 标题 + 一组按钮。`MarginSlider`（`components/builder/formatting-controls.tsx:512-538`）是 5–25mm 范围的 `<input type="range">`。`SpacingSelector`（`components/builder/formatting-controls.tsx:551-574`）渲染 1-5 数字按钮组。

底部"Effective Output"区实时显示当前设置映射的实际 CSS 值（`rem` 单位、行高、字号），帮助用户预览。

### 5.7 模板与渲染：`components/builder/template-selector.tsx` + `components/resume/`

`TemplateSelector`（`components/builder/template-selector.tsx:18`）渲染 4 个缩略图按钮。`TemplateThumbnail` 内部组件（`template-selector.tsx:80+`）用 SVG 画简单线框图代表各布局。`ResumeSingleColumn`（`components/resume/resume-single-column.tsx:29`）是最常用的"瑞士单列"模板，使用 `getSortedSections` 工具获取按 `order` 排序的可见章节（`resume-single-column.tsx:37`），再对每个 `SectionMeta` 分发渲染（`resume-single-column.tsx:93-122`）。`DynamicResumeSection`（`components/resume/dynamic-resume-section.tsx:23`）处理用户自定义章节，根据 `sectionType` 渲染 `text` / `itemList` / `stringList` 三种内容。

`SafeHtml`（`components/resume/safe-html.tsx:28`）是无 `'use client'` 指令的服务端组件，使用 `sanitizeHtml`（基于 isomorphic-dompurify）清洗 LLM 输出的 HTML，仅保留 `strong/em/u/a` 四种标签。

### 5.8 AI 重生成向导：`components/builder/regenerate-wizard.tsx` + 三个子组件

三步向导的状态机由 `RegenerateWizardStep` 联合类型定义：`idle` / `selecting` / `instructing` / `generating` / `previewing` / `complete`（`components/builder/regenerate-wizard.tsx:13-19`）。`RegenerateWizard` 是空壳调度，根据 `step` 决定打开哪个 `Dialog`（`regenerate-wizard.tsx:122-159`）：

1. `RegenerateDialog`：勾选要重生成的工作经历/项目/技能条目（`regenerate-dialog.tsx:36`）
2. `RegenerateInstructionDialog`：输入修改指令（`regenerate-instruction-dialog.tsx:37`）
3. `RegenerateDiffPreview`：对比原内容与 LLM 输出，按条目 Accept/Reject（`regenerate-diff-preview.tsx:43`）

整个状态机由外部 `useRegenerateWizard` hook 驱动（`resume-builder.tsx:160-204`），Wizard 自身只负责"打开/关闭哪个 Dialog"。

### 5.9 JD 对比视图：`components/builder/jd-comparison-view.tsx`

二分栏布局：左侧 `JDDisplay`（`components/builder/jd-display.tsx:14`，只读 JD），右侧 `HighlightedResumeView`（`components/builder/highlighted-resume-view.tsx:18`，简历中匹配 JD 关键词的部分用 `<mark className="bg-yellow-200">` 高亮）。关键词提取与统计逻辑在 `lib/utils/keyword-matcher.ts`，由父组件通过 `useMemo` 调用（`components/builder/jd-comparison-view.tsx:25,60`）。

### 5.10 Cover Letter & Outreach

`CoverLetterEditor`（`components/builder/cover-letter-editor.tsx:22`）与 `OutreachEditor`（`components/builder/outreach-editor.tsx:22`）几乎对称，都提供字数统计、保存按钮（其中 Outreach 额外提供复制到剪贴板）。`CoverLetterPreview`（`components/builder/cover-letter-preview.tsx:29`）按 `\n\n` 分段渲染信件正文 + 本地化日期。`OutreachPreview`（`components/builder/outreach-preview.tsx:15`）则使用消息气泡样式。

`GeneratePrompt`（`components/builder/generate-prompt.tsx:22`）是首次进入 Cover Letter / Outreach Tab 时的引导页。当 `isTailoredResume=false` 时提示用户先到 tailor 页生成（`generate-prompt.tsx:35-58`），否则显示"开始生成"按钮。

## 六、简历预览（`components/preview/`）

### 6.1 PaginatedPreview：`components/preview/paginated-preview.tsx`

WYSIWYG 预览器（228 行）。核心思想：**用"隐形测量 + 缩放展示"分离**——先把 `<Resume>` 渲染到一个不可见容器（`left: -9999`）以获得真实 `scrollHeight`，然后切分为多个 `<PageContainer>` 渲染（`components/preview/paginated-preview.tsx:170-225`）。

- **缩放**：0.4× 到 1.5×，默认 0.6×（`paginated-preview.tsx:18-20`）。`calculateAutoZoom` 根据容器宽度自动计算（`paginated-preview.tsx:75-82`），用户手动缩放后 `autoZoom` 置为 `false`。
- **边距辅助线**：toggle `showMargins`，渲染蓝色虚线框（`page-container.tsx:67-76`）。
- **页数显示**：右上角 `FileText` 图标 + 单复数文案（`paginated-preview.tsx:155-164`）。

### 6.2 PageContainer：`components/preview/page-container.tsx`

单页画布。`width: pageWidthPx`、`height: pageHeightPx`，`transform: scale(${scale})` 缩放（`page-container.tsx:57-65`）。`contentOffset` / `contentEnd` 让每个 PageContainer 只显示属于本页的内容片段，避免与上下页重复（`page-container.tsx:50-52`）。

### 6.3 usePagination：`components/preview/use-pagination.ts`

分页算法 hook。先 `document.fonts.ready` 等字体加载（`use-pagination.ts:53`），再 `getContentAreaPx` 算出可写区高度，遍历 `.resume-item` / `[data-no-break]` 元素确定分页点（`use-pagination.ts:71-79`），但**不**在 `.resume-section` 之间强制分页（`use-pagination.ts:70` 注释解释：让章节自然跨页）。150ms debounce 防止 resize 时频繁计算（`use-pagination.ts:34`）。

## 七、简历定制（`components/tailor/` 与 `app/(default)/tailor/page.tsx`）

### 7.1 DiffPreviewModal：`components/tailor/diff-preview-modal.tsx`

对比弹窗（440 行）。结构：
- 顶部 5 个 `StatCard` 汇总变更数（`diff-preview-modal.tsx:149-175`）：新增技能、移除技能、新增证书、修改描述、高风险变更。
- 中部按 `field_type` 分组的 `ChangeSection`（`diff-preview-modal.tsx:367-385`），可折叠。`ChangeItem`（`diff-preview-modal.tsx:392-440`）用 `+`/`-`/`~` 字符标识 `added`/`removed`/`modified`，删除线显示原值。
- 底部"Reject"和"Confirm"两个按钮，Confirm 提交后端 `confirmImproveResume`。

`elapsed` state 显示确认请求的等待秒数（`diff-preview-modal.tsx:40-54`）。弹窗关闭逻辑：`isConfirming=true` 时禁用外部关闭，避免中断正在进行的请求（`diff-preview-modal.tsx:60-64`）。

### 7.2 TailorPage 流程：`app/(default)/tailor/page.tsx`

1. 用户粘贴 JD → `uploadJobDescriptions` 提交（`tailor/page.tsx:11-15`）
2. `previewImproveResume` 异步生成预览，把结果写入 `ResumePreviewContext`（`setImprovedData`）

## 十二、关键设计模式总结

| 模式 | 体现 | 好处 |
|------|------|------|
| **受控数据流** | 所有 form 组件 `onChange` 回传整个对象 | 单一数据源，调试简单 |
| **Context 跨页面** | `ResumePreviewProvider` 串联 tailor ↔ builder | 无需 URL 编码大对象 |
| **乐观 UI + 状态机** | 多个 `useState` 标志 loading/saving/error | 反馈明确，避免重复点击 |
| **路由分发** | `Resume` 组件根据 `template` 字段选择子模板 | 4 种样式共用同一份 `ResumeData` |
| **弹窗内嵌弹窗** | `ConfirmDialog` 基于 `Dialog` 二次封装 | 语义化 API + 复用基础能力 |
| **i18n shape 强约束** | `Messages = typeof en` 强制 5 个 locale 同构 | 编译期捕获翻译漂移 |
| **a11y 优先** | `aria-modal`、`aria-labelledby`、44×44 触摸区 | 默认符合 WCAG 2.2 AA |
| **CSS Modules for 打印** | `styles/*.module.css` 隔离模板样式 | 打印 PDF 与屏幕预览共用组件 |

## 十三、学习路径建议

1. **先读 UI 层**（`ui/button.tsx` → `ui/dialog.tsx`）理解 Swiss 设计 tokens 如何落到代码。
2. **再看 Dashboard**（`dashboard/page.tsx` + `resume-upload-dialog.tsx`）理解数据获取与状态机。
3. **重点看 Builder**（`resume-builder.tsx` → `resume-form.tsx`）理解大型组件的拆分艺术。
