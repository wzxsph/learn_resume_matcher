# 01 — 项目总览

> 本章用 10 分钟带你看懂 Resume Matcher：它是什么、解决什么问题、用什么技术、各部分怎么协作。

## 一、项目定位

Resume Matcher 是一个开源的 **AI 简历定制平台**，核心场景：

> 用户上传「主简历（Master Resume）」 → 粘贴「职位描述（JD）」 → AI 分析差距 → 生成定制化简历建议 → 用户编辑 → 导出 PDF + 求职信。

与之类似的工具有 Resume Worded、Jobscan、Teal 等，但 Resume Matcher 的差异化是 **本地可跑**（用 Ollama）+ **完全开源** + **支持多家 LLM**。

参考：[README.zh-CN.md:11](https://github.com/srbhr/Resume-Matcher/blob/main/README.zh-CN.md#L11)

## 二、核心能力一览

| 能力 | 描述 | 入口页面 |
|------|------|----------|
| **主简历管理** | 一份完整简历作为「单一事实源」 | `apps/frontend/app/(default)/dashboard` |
| **JD 匹配** | AI 分析 JD 与简历差距，输出定制建议 | `components/tailor/` |
| **简历改进** | 量化、可操作的简历内容改写 | `components/builder/` |
| **AI 富集** | 多轮对话式补全简历缺失信息 | `components/enrichment/` |
| **求职信生成** | 基于简历 + JD 生成求职信 | `apps/backend/app/services/cover_letter.py` |
| **PDF 导出** | Playwright + Chromium 渲染多种模板 | `apps/backend/app/pdf.py` |
| **国际化** | 界面支持 5 种语言 | `apps/frontend/messages/` |

参考：[README.zh-CN.md:101-148](https://github.com/srbhr/Resume-Matcher/blob/main/README.zh-CN.md#L101-L148)

## 三、用户工作流

整个产品的用户旅程（参考 [docs/agent/architecture/frontend-workflow.md](https://github.com/srbhr/Resume-Matcher/blob/main/docs/agent/architecture/frontend-workflow.md)）：

```
1. Settings: 配置 AI 提供商（Ollama / OpenAI / Anthropic / Gemini …）
2. Upload: 上传主简历（PDF / DOCX）
3. Tailor: 粘贴 JD → AI 输出改进建议
4. Builder: 增删章节、拖拽排序、切换模板
5. Enrichment: AI 多轮补全薄弱章节
6. Cover Letter: 生成求职信
7. Export: 选模板 → 下载 PDF
```

## 四、技术栈全景

### 后端 — `apps/backend/`

| 技术 | 用途 | 关键文件 |
|------|------|----------|
| **FastAPI** | HTTP 框架，自动 OpenAPI 文档 | `app/main.py` |
| **Python 3.13+** | 现代类型系统、模式匹配 | `pyproject.toml` |
| **LiteLLM** | 统一多 LLM provider 接口 | `app/llm.py` |
| **TinyDB** | JSON 文件型数据库（无外部依赖） | `app/database.py` |
| **Pydantic v2** | 数据校验与序列化 | `app/schemas/` |
| **Playwright** | 无头 Chromium 渲染 PDF | `app/pdf.py` |
| **uv** | 极速 Python 依赖管理 | `pyproject.toml` |

### 前端 — `apps/frontend/`

| 技术 | 用途 | 关键文件 |
|------|------|----------|
| **Next.js 16** | App Router + Turbopack | `next.config.ts` |
| **React 19** | Server Components + Actions | `app/` |
| **TypeScript 5** | 类型安全 | `tsconfig.json` |
| **Tailwind CSS v4** | 原子化样式 | `postcss.config.mjs` |
| **Tiptap** | 富文本编辑（简历内容） | `components/builder/` |
| **dnd-kit** | 拖拽排序（章节调整） | `package.json` |
| **Lucide Icons** | 图标库 | `package.json` |
| **Vitest** | 单元测试 | `vitest.config.ts` |
| **Swiss Design** | 设计系统（硬边、衬线、橙红强调） | `docs/portable/swiss-design-system/` |

参考：[apps/frontend/package.json:1-40](https://github.com/srbhr/Resume-Matcher/blob/main/apps/frontend/package.json#L1-L40)

## 六、AI 集成亮点

Resume Matcher 通过 [LiteLLM](https://github.com/srbhr/Resume-Matcher/blob/main/apps/backend/app/llm.py) 抽象了 6 家 LLM provider：

| Provider | 用途 | 配置方式 |
|----------|------|----------|
| **Ollama** | 本地免费 | `OPENAI_BASE_URL=http://localhost:11434/v1` |
| **OpenAI** | GPT-4o / GPT-4o-mini | `OPENAI_API_KEY=sk-...` |
| **Anthropic** | Claude 3.5 Sonnet | `ANTHROPIC_API_KEY=sk-ant-...` |
| **Google Gemini** | Gemini 1.5 Flash/Pro | `GEMINI_API_KEY=...` |
| **OpenRouter** | 多模型聚合 | `OPENROUTER_API_KEY=...` |
| **DeepSeek** | DeepSeek Chat | `DEEPSEEK_API_KEY=...` |

参考：[README.zh-CN.md:200-212](https://github.com/srbhr/Resume-Matcher/blob/main/README.zh-CN.md#L200-L212)

## 七、设计哲学 — Swiss International Style

整个 UI 严格遵循 **瑞士国际主义平面设计**（参考 [docs/portable/swiss-design-system/README.md](https://github.com/srbhr/Resume-Matcher/blob/main/docs/portable/swiss-design-system/README.md)）：

- **画布**: `#F0F0E8`（米白）
- **墨色**: `#000000`（纯黑文字）
- **超链接蓝**: `#1D4ED8`
- **信号绿**: `#15803D`（成功）
- **警告橙**: `#F97316`
- **错误红**: `#DC2626`
- **标题**: `font-serif`（衬线）
- **正文**: `font-sans`（无衬线）
- **元数据**: `font-mono`（等宽）
- **边框**: `rounded-none` 硬边 + 1px 黑色 + 硬阴影

参考：[.claude/CLAUDE.md:108-117](https://github.com/srbhr/Resume-Matcher/blob/main/.claude/CLAUDE.md#L108-L117)

## 八、关键非协商规则（来自项目 CLAUDE.md）

1. 前端 UI 变更必须遵循 Swiss 设计系统
2. 所有 Python 函数必须有类型提示
3. 提交前端代码前必须运行 `npm run lint` 和 `npm run format`
4. 详细错误日志在服务端，客户端只返回通用消息
5. **不修改** `.github/workflows/` 文件（除非显式要求）

参考：[.claude/CLAUDE.md:17-22](https://github.com/srbhr/Resume-Matcher/blob/main/.claude/CLAUDE.md#L17-L22)
