# 03 · Resume Matcher 后端架构总览

> 适用版本：`apps/backend/pyproject.toml:3`（`version = "1.2.0"`）。本文面向学习 FastAPI 项目的中级开发者。

## 一、目标与定位

Resume Matcher 后端是一个**本地优先（local-first）** 的 FastAPI 服务，核心职责是把用户的简历与岗位描述（JD）交给 LLM 进行"定向改写"，并把改写结果以结构化 JSON、Markdown 文本、PDF 三种形式返回。整个后端不依赖外部数据库，所有持久化数据存放在 `apps/backend/data/` 目录下的 JSON 文件里，可一键重置。

主要技术栈（见 `apps/backend/pyproject.toml:6-19`）：

| 组件 | 选型 | 作用 |
|------|------|------|
| Web 框架 | `fastapi==0.128.4` | 路由、参数校验、OpenAPI 文档 |
| ASGI 服务器 | `uvicorn==0.40.0` | 启动 HTTP 服务 |
| 配置 | `pydantic==2.12.5` + `pydantic-settings==2.12.0` | 环境变量与 `.env` 解析 |
| 数据库 | `tinydb==4.8.2` | JSON 文件存储 |
| AI 抽象层 | `litellm==1.86.2` | 多 Provider LLM 统一接口 |
| 文档解析 | `markitdown[docx]==0.1.4` | PDF/DOCX → Markdown |
| 文档输出 | `pdfminer.six==20260107` | PDF 文本抽取 |
| 浏览器 | `playwright==1.58.0` | 无头 Chromium 渲染 PDF |
| 依赖管理 | `uv` | Python 包与虚拟环境 |

### 3.1 日志配置


### 3.2 lifespan

`main.py:33-50` 使用 `@asynccontextmanager` 实现 FastAPI 的现代 lifespan 协议：

- **启动**：确保 `settings.data_dir` 存在（`main.py:37`）。注意：PDF 渲染器**采用懒加载**，首次调用 `render_resume_pdf` 才会启动 Chromium，注释行 `main.py:39` 明确写出了这一点。
- **关闭**：分两个 `try/except` 关闭 PDF 渲染器与数据库（`main.py:42-50`），保证任何一个失败都不会影响另一个的清理。

### 4.2 API Key 优先级链

`config.py:41-112` 与 `llm.py:323-349` 共同实现了"**API Key 单一真源**"：

```
Settings.llm_api_key  (环境变量 / .env)
        ↓ 若为空
config.json 顶层 api_key
        ↓ 若为空
config.json  api_keys[provider]   (多 provider 形态)
        ↓ 若为空
settings.llm_api_key   ← 关键：openai_compatible / ollama 会跳过这一步
```

`openai_compatible` 与 `ollama` 被刻意排除在"env 兜底"之外（`llm.py:318-320`），以免付费的 `OPENAI_API_KEY` 泄漏到本地 LLM。

`Database` 类自身并不在 `__init__` 里打开文件，而是在 `db` 属性里懒加载（`database.py:28-33`）。这意味着只要不调用数据库方法，启动服务时就不会持有文件句柄。

### 5.3 Master 简历并发控制

`database.py:21` 定义了一个**类级** `asyncio.Lock`：

```python
_master_resume_lock = asyncio.Lock()
```

`create_resume_atomic_master`（`database.py:98-139`）在锁内完成：
1. 读取当前 master。
2. 若 master 处于 `failed` / `processing`，先降级再把新简历晋升为 master。
3. 调用 `create_resume` 写入。

`apps/backend/CLAUDE.md` 明确指出锁使用 `asyncio.Lock` 而非 `threading.Lock`，避免阻塞事件循环。

### 6.1 Router 与配置指纹

`llm.py:455-526` 用 `litellm.Router` 把单模型封装成可重试的传输层，键控条件是 `_config_fingerprint`（`llm.py:460-469`）—— 只要 `provider | model | hash(api_key) | api_base` 不变就复用同一个 Router。`num_retries=3` 的 `RetryPolicy`（`llm.py:492-499`）区分错误类型：对鉴权失败和 4xx 不重试，对超时/限流/5xx 才重试。

### 6.2 三个公开异步函数

| 函数 | 位置 | 用途 |
|------|------|------|
| `check_llm_health` | `llm.py:529-648` | 用一句 `"Hi"` 探活，并把错误信息脱敏后再返回 |
| `complete` | `llm.py:651-699` | 纯文本补全，自动剥离 `<think>` 标签 |
| `complete_json` | `llm.py:1026-1156` | 结构化补全，JSON 模式 → prompt-only 回退、3 次重试、截断感知 |

### 6.3 自适应超时

`llm.py:897-924` 的 `_calculate_timeout` 同时考虑"操作类型"与"Provider 因子"：

```python
provider_factors = {
    "openai": 1.0,
    "anthropic": 1.2,
    "openrouter": 1.5,
    "groq": 1.0,
    "ollama": 2.0,  # 本地模型最慢
}
```

基础值在 `llm.py:39-41`：`health_check=30s`、`completion=120s`、`json=180s`，再按 `max_tokens/4096` 比例放大。

### 6.4 JSON 提取与截断检测

`_extract_json`（`llm.py:941-1023`）用大括号配对算法从 Markdown 代码块 / 原始 JSON 中切出字符串；`_appears_truncated`（`llm.py:781-824`）对 `resume` / `enrichment` / `diff` / `keywords` 四类 schema 各自判断是否可能被截断，命中时 `complete_json` 会把 prompt 追加提示并重试。

## 七、PDF 渲染

`apps/backend/app/pdf.py` 通过 Playwright 启动无头 Chromium，把前端 `/print/*` 页面渲染成 PDF。

### 7.1 懒加载与平台兼容

```python
# apps/backend/app/pdf.py:26-30
_playwright = None
_browser: Optional[Browser] = None
_init_lock = asyncio.Lock()

```

`init_pdf_renderer`（`pdf.py:33-51`）使用 `asyncio.Lock` 防止并发初始化。`render_resume_pdf`（`pdf.py:251-310`）在事件循环不支持子进程（Windows 的 SelectorEventLoop）时自动退化到 `_render_resume_pdf_in_thread`，因此在 `main.py:11-12` 还设置了 `WindowsProactorEventLoopPolicy`。

### 7.2 浏览器查找

`_find_chromium_executable`（`pdf.py:73-111`）在 Windows / macOS / Linux 上分别扫描 Chrome / Edge / Chromium / Snap / Flatpak 的常见安装路径；找不到则用 `PDFRenderError` 提示用户执行 `python -m playwright install chromium`（`pdf.py:209-227`）。

### 7.3 渲染管线

```python
# apps/backend/app/pdf.py:129-143
async def _render_page_to_pdf(page, url, selector, pdf_format, pdf_margins):
    await page.goto(url, wait_until="networkidle")
    await page.wait_for_selector(selector)
    await page.evaluate("document.fonts.ready")
    return await page.pdf(format=pdf_format, print_background=True, margin=pdf_margins)
```

等待 CSS 选择器（默认 `.resume-print`）出现且字体加载完成后再调用 `page.pdf`，避免出现空白页或回退字体。

## 十二、关键约定

- **类型注解**：所有 Python 函数（包含私有 helper）都必须带类型注解（`apps/backend/CLAUDE.md` Non-Negotiable Rules #1）。
- **错误处理**：服务端记详细日志，客户端只回泛化消息（参考 `main.py` / `resumes.py:120-127` 的 `_raise_improve_error`）。
- **不可变共享对象**：所有"可变默认值"与"缓存读出"必须 `copy.deepcopy()`（参考 `models.py:326-338` 的 `normalize_resume_data`）。
- **路由挂载**：新端点必须挂在 `/api/v1` 前缀下，并在 `routers/__init__.py` 暴露。
- **单 worker 假设**：所有 `asyncio.Lock` / 全局缓存（`config_cache`、`db`）均假设只有一个 uvicorn worker；多 worker 部署需要先重审这些并发原语。

## 十三、小结

- 后端只有 6 个核心文件（`main.py` / `config.py` / `database.py` / `llm.py` / `pdf.py` + 一个 schema 总入口），其余都是"路由-服务"两层扩展。
- **路由层**只做参数校验、流程编排、超时控制；**服务层**不直接碰数据库；**LLM 抽象层**统一处理 Provider 差异、JSON 解析、重试、截断。
- 配置与 API key 走"env → config.json → 多 provider 字典"三级回退，且对本地 Provider 刻意跳过 env 兜底。
- 数据全部本地化（`apps/backend/data/`），可一键 `POST /api/v1/config/reset` 销毁（`config.py:568-594`），但会要求传 `{"confirm": "RESET_ALL_DATA"}` 二次确认。
