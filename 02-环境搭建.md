# 02 — 环境搭建

> 本章手把手带你把 Resume Matcher 在本地跑起来，覆盖 uv 依赖、Node 模块、AI provider 配置、Docker 三种方式。

## 一、前置条件

| 工具 | 最低版本 | 检查命令 | 安装 |
|------|----------|----------|------|
| **Python** | 3.13+ | `python --version` | [python.org](https://python.org) |
| **Node.js** | 22+ | `node --version` | [nodejs.org](https://nodejs.org) |
| **npm** | 10+ | `npm --version` | 随 Node 一起 |
| **uv** | 最新 | `uv --version` | [astral.sh/uv](https://docs.astral.sh/uv/getting-started/installation/) |
| **Git** | 任意 | `git --version` | [git-scm.com](https://git-scm.com) |

参考：[SETUP.zh-CN.md:36-40](https://github.com/srbhr/Resume-Matcher/blob/main/SETUP.zh-CN.md#L36-L40)

### 安装 uv（Python 包管理器）

项目使用 `uv`（Astral 出品的 Rust 写成的极速 pip 替代品）：

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows (PowerShell)
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 或者用 pip
pip install uv
```

参考：[SETUP.zh-CN.md:44-55](https://github.com/srbhr/Resume-Matcher/blob/main/SETUP.zh-CN.md#L44-L55)

## 二、快速开始（最快 5 分钟）

如果你只关心能不能跑：

```bash
# 1. 克隆
git clone https://github.com/srbhr/Resume-Matcher.git
cd Resume-Matcher

# 2. 后端（终端 1）
cd apps/backend
cp .env.example .env        # 必须复制，否则 LLM 调用会失败
uv sync                      # 安装 Python 依赖（首次约 30s）
uv run app                   # 启动 FastAPI，默认 :8000

# 3. 前端（终端 2）
cd apps/frontend
npm install                  # 安装 Node 依赖（首次约 1-2 min）
npm run dev                  # 启动 Next.js，默认 :3000
```

打开 **<http://localhost:3000>** 即可。但你还需要在 Settings 里配置 AI provider 才能让 AI 工作。

参考：[SETUP.zh-CN.md:62-83](https://github.com/srbhr/Resume-Matcher/blob/main/SETUP.zh-CN.md#L62-L83)

## 三、详细步骤

### 步骤 1：克隆仓库

```bash
git clone https://github.com/srbhr/Resume-Matcher.git
cd Resume-Matcher
```

### 步骤 2：后端配置

#### 2.1 进入后端目录

```bash
cd apps/backend
```

#### 2.2 创建环境变量文件

```bash
cp .env.example .env
```

> **这一步不可省略！** 没有 `.env` 文件，LiteLLM 不知道用哪个 provider。

#### 2.3 编辑 `.env`

最少需要配置一个 provider（详见下节「AI provider 配置」）。

### 步骤 3：前端配置

```bash
cd apps/frontend
npm install
npm run dev
```

`npm install` 会读取 `package.json` 安装所有依赖（参考 [apps/frontend/package.json:1-40](https://github.com/srbhr/Resume-Matcher/blob/main/apps/frontend/package.json#L1-L40)）。

`npm run dev` 实际执行 `next dev --turbopack`（参考 [package.json:6](https://github.com/srbhr/Resume-Matcher/blob/main/apps/frontend/package.json#L6)），用 Turbopack 启动开发服务器。

## 四、配置 AI Provider

### 选项 A：本地免费 — Ollama（推荐新手）

1. 安装 Ollama: [ollama.com](https://ollama.com)
2. 拉取一个模型（首次约 5GB）：
   ```bash
   ollama pull llama3.2
   ollama run llama3.2  # 测试一下
   ```
3. 启动 Ollama 服务（默认监听 `11434`）
4. 配置 `.env`：
   ```env
   LLM_PROVIDER=ollama
   LLM_MODEL=llama3.2
   LLM_BASE_URL=http://localhost:11434
   ```
   或者用 OpenAI 兼容模式：
   ```env
   LLM_PROVIDER=openai
   LLM_MODEL=llama3.2
   OPENAI_API_KEY=ollama
   OPENAI_BASE_URL=http://localhost:11434/v1
   ```

参考：[SETUP.zh-CN.md:160-200](https://github.com/srbhr/Resume-Matcher/blob/main/SETUP.zh-CN.md#L160-L200)

### 选项 B：云端 OpenAI

```env
LLM_PROVIDER=openai
LLM_MODEL=gpt-4o-mini
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.openai.com/v1
```

### 选项 C：Anthropic Claude

```env
LLM_PROVIDER=anthropic
LLM_MODEL=claude-3-5-sonnet-20241022
ANTHROPIC_API_KEY=sk-ant-...
```

### 选项 D：Google Gemini

```env
LLM_PROVIDER=gemini
LLM_MODEL=gemini-1.5-flash
GEMINI_API_KEY=...
```

### 选项 E：OpenRouter / DeepSeek

类似配置，详细参数参考 [SETUP.zh-CN.md](https://github.com/srbhr/Resume-Matcher/blob/main/SETUP.zh-CN.md)。

## 五、Docker 部署

### 方式 1：拉取官方镜像

```bash
docker pull srbhr/resume-matcher:latest
docker run -p 3000:3000 -p 8000:8000 srbhr/resume-matcher:latest
```

参考：[README.zh-CN.md:215-220](https://github.com/srbhr/Resume-Matcher/blob/main/README.zh-CN.md#L215-L220)

### 方式 2：本地 docker-compose

```bash
# 在仓库根目录
docker-compose up
```

`docker-compose.yml` 会同时启动 backend 和 frontend，共享网络。

### Docker 中用 Ollama

如果 Ollama 跑在宿主机，Docker 容器内要访问它：

```env
LLM_BASE_URL=http://host.docker.internal:11434
```

> **不要用 `localhost`**，容器内 `localhost` 指向容器自己。

参考：[README.zh-CN.md:222](https://github.com/srbhr/Resume-Matcher/blob/main/README.zh-CN.md#L222)

## 八、首次使用流程

1. 打开 `http://localhost:3000`
2. 进入 **Settings** 页面
3. 选择 AI provider（如果 `.env` 没配好，前端会读取空配置）
4. 上传主简历（PDF / DOCX）
5. 进入 **Tailor**，粘贴 JD，点击「生成建议」
6. 在 **Builder** 编辑章节、选择模板
7. 导出 PDF

## 九、故障排查

### 后端启动报错：`ModuleNotFoundError`

```bash
# 解决：重新同步依赖
cd apps/backend
uv sync
```

### 前端 `npm install` 慢 / 失败

```bash
# 换镜像源
npm config set registry https://registry.npmmirror.com
npm install
```

### LLM 调用失败：`401 Unauthorized`

- 检查 `.env` 里的 `*_API_KEY` 是否正确
- 检查 `LLM_PROVIDER` 与 key 是否匹配

### Ollama 连接不上

```bash
# 确认 Ollama 在运行
curl http://localhost:11434/api/tags

# Docker 场景：用 host.docker.internal
LLM_BASE_URL=http://host.docker.internal:11434
```

### PDF 导出失败：Playwright 报错

```bash
# 安装 Chromium
cd apps/backend
uv run playwright install chromium
```

参考：[apps/backend/app/pdf.py](https://github.com/srbhr/Resume-Matcher/blob/main/apps/backend/app/pdf.py) 中 Playwright 初始化逻辑。
