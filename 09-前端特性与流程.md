# 09 - 前端特性与流程

> 本章面向希望理解 Resume Matcher 前端特性实现与典型用户流程的开发者。我们将依次剖析国际化、自定义章节、简历模板、AI 富集、JD 匹配等核心模块，并梳理从打开站点到导出 PDF 的端到端流程。

---

## 1. 技术栈与目录约定

前端位于 `apps/frontend/`，技术栈为 **Next.js 16（App Router + Turbopack）**、**React 19**、**TypeScript（strict）**、**Tailwind CSS v4**。所有路由入口位于 `app/`，UI 组件位于 `components/`，后端调用集中在 `lib/api/`，自定义 Hooks 位于 `hooks/`，消息翻译文件位于 `messages/`，i18n 引擎位于 `lib/i18n/` 与 `i18n/config.ts`。

> 详细的目录约定与文件分工可参考 `apps/frontend/CLAUDE.md:23-58`。

## 2. 国际化（i18n）

### 2.1 两套语言设置

Resume Matcher 在 Settings 页面维护两套独立语言：

- **UI 语言（uiLanguage）**：界面文案，仅在 localStorage 持久化，影响按钮、菜单、提示文本。
- **内容语言（contentLanguage）**：LLM 生成内容（简历、求职信）使用的语言，既写 localStorage，也会同步到后端配置。

两套语言相互独立，例如界面用中文而生成英文简历是支持的（见 `docs/agent/features/i18n.md:17-23`）。

### 2.2 文件结构

```
apps/frontend/
├── i18n/
│   └── config.ts             # locales / localeNames / localeFlags（来源真相）
├── lib/i18n/
│   ├── index.ts              # 桶导出
│   ├── messages.ts           # 静态 import 全部 JSON；导出 Messages 类型
│   ├── translations.ts       # useTranslations hook（客户端）
│   ├── server.ts             # translate()（服务端）
│   ├── locale.ts             # resolveLocale
│   └── utils.ts              # getNestedValue / applyParams
├── messages/
│   ├── en.json               # 来源真相，Messages 类型从此推导
│   ├── es.json
│   ├── zh.json
│   ├── ja.json
│   └── pt-BR.json            # 文件名是 pt-BR，但 i18n/config.ts 把它注册为 pt
└── lib/context/
    └── language-context.tsx  # LanguageProvider，UI + Content 两层状态
```

源码真相位于 `apps/frontend/i18n/config.ts:5-25`，五种支持的语言为 `en / es / zh / ja / pt`（其中 `pt` 的实际文件是 `messages/pt-BR.json`，由 `lib/i18n/messages.ts:7` 导入）。

### 2.3 翻译机制

**客户端组件**使用 `useTranslations()` Hook（`apps/frontend/lib/i18n/translations.ts:16-37`）：

```typescript
const { t } = useTranslations();
<button>{t('common.save')}</button>
```

`t()` 接受点号分隔的 key，从 `messages/{locale}.json` 的嵌套结构中取值，并通过 `applyParams`（`apps/frontend/lib/i18n/utils.ts:16-24`）替换 `{placeholder}` 形式的占位符。

**服务端组件**（如 `app/print/resumes/[id]/page.tsx`）使用 `translate(locale, key, params)`（`apps/frontend/lib/i18n/server.ts:5-13`）。

**类型推导**的关键在于 `apps/frontend/lib/i18n/messages.ts:9`：

```typescript
export type Messages = typeof en;
const allMessages: Record<Locale, Messages> = { en, es, zh, ja, pt };
```

> ⚠️ **构建中断陷阱**：因为 `Messages` 是从 `en.json` 推导的严格结构类型，所有其它语言的 JSON 必须与 `en.json` **结构完全对齐**。给 `en.json` 增删 key 后必须同步修改其余四个文件，否则 `npm run build` 会失败（见 `apps/frontend/CLAUDE.md:115-122`）。

### 2.4 添加新语言（实操步骤）

1. **复制 `en.json`** 为新语言文件，例如 `apps/frontend/messages/de.json`，翻译每一项 value（保持 key 路径不变）。
2. **在 `apps/frontend/i18n/config.ts`** 中追加 locale：
   ```typescript
   export const locales = ['en', 'es', 'zh', 'ja', 'pt', 'de'] as const;
   export const localeNames: Record<Locale, string> = { ..., de: 'Deutsch' };
   export const localeFlags: Record<Locale, string> = { ..., de: '🇩🇪' };
   ```
3. **在 `apps/frontend/lib/i18n/messages.ts`** 中新增 import 与映射：
   ```typescript
   import de from '@/messages/de.json';
   const allMessages: Record<Locale, Messages> = { en, es, zh, ja, pt, de };
   ```
4. **在 `apps/backend/app/prompts/templates.py`** 的 `LANGUAGE_NAMES` 与 `get_language_name()` 中追加语言代码到全称的映射，确保 LLM 能正确接收 `{output_language}`。
5. **运行** `npm run build` 验证所有 JSON 结构对齐（构建器会做类型检查）。

## 3. 自定义章节（Custom Sections）

### 3.1 章节类型

详见 `docs/agent/features/custom-sections.md:5-13`：

| Type | 描述 | 适用场景 |
|------|------|---------|
| `personalInfo` | 特殊类型，固定在头部 | 姓名、联系方式 |
| `text` | 单一文本块 | Summary、Objective、Statement |
| `itemList` | 数组型条目 | Experience、Projects、Publications |
| `stringList` | 简单字符串数组 | Skills、Languages、Hobbies |

### 3.2 章节控制

每个非 `personalInfo` 章节的 header 处提供五个控件（`docs/agent/features/custom-sections.md:24-33`）：

- **可见性（Eye / EyeOff）**：切换是否在 PDF 中显示。隐藏章节仍可在表单中编辑，仅在 PDF/预览中过滤（`getSortedSections` 与 `getAllSections` 的差别）。
- **上移/下移（ChevronUp/ChevronDown）**：调整章节顺序。
- **重命名（Pencil）**：就地编辑显示名。
- **删除（Trash）**：默认隐藏，自定义章节可彻底删除。

### 3.3 数据结构

```typescript
interface ResumeData {
  // ... 内建字段
  sectionMeta?: SectionMeta[];                          // 顺序、显示名、可见性
  customSections?: Record<string, CustomSection>;       // 用户自定义章节
}
```

章节元数据定义在 `apps/backend/app/schemas/models.py`（`SectionType / SectionMeta / CustomSection`），状态由 `apps/frontend/lib/utils/section-helpers.ts` 维护。

### 3.4 关键实现细节

- **隐藏视觉反馈**：隐藏章节在表单里显示为虚线边框 + 60% 透明度，并附带琥珀色 "Hidden from PDF" 徽章（`docs/agent/features/custom-sections.md:36-41`）。
- **迁移兼容**：旧数据通过懒加载归一化（lazy normalization）补齐 `sectionMeta`，无需手工迁移。
- **不可变默认值**：使用 `copy.deepcopy(DEFAULT_SECTION_META)` 避免共享可变引用 bug（见 `docs/agent/features/custom-sections.md:68`，与 `apps/backend/CLAUDE.md:130-132` 强调一致）。

### 3.5 添加新章节（步骤）

1. 在 `apps/frontend/components/builder/add-section-dialog.tsx` 的章节类型选项中暴露新类型（如需）。
2. 编写对应的表单组件（参考 `forms/` 目录下既有的 `generic-text-form.tsx` / `generic-item-form.tsx` / `generic-list-form.tsx`）。
3. 在 `apps/frontend/components/resume/dynamic-resume-section.tsx` 中处理该类型在 PDF 模板中的渲染。
4. 视情况扩展 `apps/backend/app/schemas/models.py` 的 `SectionType` 联合类型。

## 4. 简历模板系统

### 4.1 现有模板

模板类型定义在 `apps/frontend/lib/types/template-settings.ts`（`TemplateType`）。当前四种模板见 `docs/agent/features/resume-templates.md:5-13`：

| Template | 描述 |
|----------|------|
| `swiss-single` | 单列，最大内容密度，1-2 页首选 |
| `swiss-two-column` | 65%/35% 双列，左侧经历，右侧技能 |
| `modern` | 单列 + 彩色 accent header，可换主题色 |
| `modern-two-column` | modern + 双列布局 |

四种模板文件位于 `apps/frontend/components/resume/`：

```

└── styles/

    ├── _base.module.css

```

### 4.2 格式化控件

`TemplateSettings` 提供以下调参（详见 `docs/agent/features/resume-templates.md:15-30`）：

| 控件 | 范围 | 默认 | 作用 |
|------|------|------|------|
| Margins | 5-25mm | 8mm | 页面边距 |
| Section Spacing | 1-5 | 3 | 章节间距 |
| Item Spacing | 1-5 | 2 | 条目间距 |
| Line Height | 1-5 | 3 | 行高 |
| Base Font Size | 1-5 | 3 | 基础字号缩放（11-16px） |
| Header Scale | 1-5 | 3 | 标题缩放 |
| Header Font | serif / sans-serif / mono | serif | 标题字体 |
| Body Font | serif / sans-serif / mono | sans-serif | 正文字体 |
| Compact Mode | bool | false | 0.6x 间距（仅间距） |
| Contact Icons | bool | false | 联系信息加图标 |
| Accent Color | blue/green/orange/red | blue | modern 系列 accent 颜色 |

### 4.3 CSS 变量机制

模板通过 CSS 自定义属性驱动排版，根样式位于 `apps/frontend/components/resume/styles/_tokens.css`：

```css
--section-gap, --item-gap, --line-height
--font-size-base, --header-scale, --section-header-scale
--header-font, --body-font
--margin-top / bottom / left / right
--accent-primary, --accent-light
```

新模板必须使用 `baseStyles['resume-section']` / `baseStyles['resume-item-subtitle']` 等共享类（`docs/agent/features/resume-templates.md:54-55`）以响应设置变化。

### 4.4 添加新模板（步骤）

参考 `docs/agent/features/adding-resume-templates.md`：

1. **创建组件** `components/resume/resume-{name}.tsx`：
   ```tsx
   interface TemplateProps {
     resumeData: ResumeData;
     settings: TemplateSettings;
   }
   export function ResumeNewTemplate({ resumeData, settings }: TemplateProps) {
     return (
       <div className="resume-print">
         <header className="resume-section">
           <h1>{resumeData.personalInfo?.name}</h1>
         </header>
         {getSortedSections(resumeData).map(section => (
           <section key={section.id} className="resume-section">
             <h3 className="resume-section-title">{section.displayName}</h3>
             <div className="resume-items">...</div>
           </section>
         ))}
       </div>
     );
   }
   ```
2. **导出**：`components/resume/index.ts` 增补 `export { ResumeNewTemplate }`。
3. **注册到选择器**：`components/builder/formatting-controls.tsx` 的 `TEMPLATES` 数组中追加新模板 id。
4. **添加缩略图**：在选择器 UI 上提供预览。
5. **测试**：跨多种 section 类型与 2 页以上内容验证 PDF 分页。

### 4.5 必需的 CSS 类

新模板必须使用以下类（Playwright 等待 `.resume-print`、`.resume-item` 不跨页，见 `docs/agent/design/pdf-template-guide.md:78-85`）：

```css
.resume-print          /* 根容器，Playwright 等待 */
.resume-section        /* 章节容器 */
.resume-section-title  /* 章节标题 */
.resume-items          /* 条目容器 */
.resume-item           /* 单条目（禁止分页） */
```

## 5. AI 富集（Enrichment）

### 5.1 工作流

`docs/agent/features/enrichment.md:7-19` 描述了一个 5 步闭环：

1. 用户在 master resume 页面点击 **Enhance Resume**。
2. AI 分析简历，识别薄弱/泛化的描述。
3. 系统最多生成 6 个针对性强的问题（防止用户疲劳）。
4. 用户回答后，AI 生成新的 bullet points。
5. 新 bullet points **追加**到原内容之后（不替换原内容）。

### 5.2 API 接口

| 端点 | 说明 |
|------|------|
| `POST /enrichment/analyze/{resume_id}` | 分析简历并生成问题 |
| `POST /enrichment/enhance` | 根据用户答案生成增强描述 |
| `POST /enrichment/apply/{resume_id}` | 应用增强到简历 |

源码分布：
- 后端提示词：`apps/backend/app/prompts/enrichment.py`（`ANALYZE_RESUME_PROMPT` / `ENHANCE_DESCRIPTION_PROMPT` / `REGENERATE_ITEM_PROMPT` / `REGENERATE_SKILLS_PROMPT`）。
- 后端路由：`apps/backend/app/routers/enrichment.py`。
- 前端状态机：`apps/frontend/hooks/use-enrichment-wizard.ts`。
- 前端 UI：`apps/frontend/components/enrichment/`。

### 5.3 设计决策（`docs/agent/features/enrichment.md:23-26`）

- **最多 6 题**：权衡用户耐心与信息覆盖度。
- **叠加式增强**：原始 bullet 保留，新内容追加。
- **优先级**：AI 挑选 ROI 最高的问题。

## 6. JD 匹配（JD Match）

### 6.1 触发条件

只有当一份简历是**针对特定 JD 量身定制**时，Builder 中才会出现 **JD MATCH** 标签。普通 master resume 不会暴露此视图（`docs/agent/features/jd-match.md:9-13`）。

### 6.2 视图结构

`docs/agent/features/jd-match.md:13-17` 描述的布局：

- **左面板**：原始 JD（只读）。
- **右面板**：简历视图，命中关键词高亮黄色。

### 6.3 匹配规则

`docs/agent/features/jd-match.md:21-25`：

- 关键词提取时过滤停用词。
- 大小写不敏感匹配。
- 统计总关键词、命中数、命中率。
- 命中率颜色编码：**≥50% 绿、≥30% 黄、<30% 红**。

### 6.4 关键实现

- 关键词提取：`apps/frontend/lib/utils/keyword-matcher.ts`。
- 视图组件：`apps/frontend/components/builder/jd-comparison-view.tsx`、`jd-display.tsx`、`highlighted-resume-view.tsx`。
- 后端接口：`apps/backend/app/routers/resumes.py` 中 `GET /{resume_id}/job-description` 用于取回当初 tailor 时使用的 JD。

## 7. Swiss International Style 设计系统

> Resume Matcher 的所有 UI **必须**遵循 Swiss International Style。完整规范位于 `docs/portable/swiss-design-system/`，本节摘录关键点。

### 7.1 核心原则

见 `docs/portable/swiss-design-system/README.md:21-28`：

1. **Grid-based layouts** — 数学精度而非直觉。
2. **Asymmetric balance** — 留白策略而非居中对称。
3. **Objective typography** — serif 标题、mono 元数据、sans 正文。
4. **Minimal ornamentation** — 硬边、无渐变、无圆角、无装饰图标。

> 一句话记忆：**square corners, hard shadows, pure colors, three-font hierarchy**。

### 7.2 设计 Token

#### 颜色（`docs/portable/swiss-design-system/tokens.md:9-22`）

| Name | Hex | 用途 |
|------|-----|------|
| Canvas | `#F0F0E8` | 页面背景（暖白，绝不用纯白） |
| Ink | `#000000` | 文本、边框 |
| Hyper Blue | `#1D4ED8` | 链接、主操作、focus ring |
| Signal Green | `#15803D` | 成功、确认、下载 |
| Alert Orange | `#F97316` | 警告 |
| Alert Red | `#DC2626` | 错误、销毁操作 |
| Steel Grey | `#4B5563` | 次级文字、禁用态 |

#### 字体（`docs/portable/swiss-design-system/tokens.md:39-43`）

```
font-serif  → 标题（Georgia / Times）
font-sans   → 正文（Inter / Helvetica / system-ui）
font-mono   → 元数据、标签（SF Mono / Consolas）
```

#### 字号（`docs/portable/swiss-design-system/tokens.md:60-66`）

```
xs(12)  sm(14)  base(16)  lg(18)  xl(20)  2xl(24)  3xl(30)  4xl(36)  5xl(48)
```

#### 阴影（`docs/portable/swiss-design-system/tokens.md:96-106`）

**只允许硬阴影**（无 blur）：

```css
shadow-[2px_2px_0px_0px_#000000]  /* 按钮 */
shadow-[4px_4px_0px_0px_#000000]  /* 卡片 */
shadow-[8px_8px_0px_0px_#000000]  /* hero */
```

hover 行为：硬阴影元素 "按下去"，即 `hover:translate-y-[1px] hover:translate-x-[1px] hover:shadow-none`（`docs/portable/swiss-design-system/tokens.md:109-115`）。

### 7.3 组件范例（`docs/portable/swiss-design-system/components.md:9-26`）

```jsx
<button className="
  rounded-none
  border-2 border-black
  bg-blue-700 text-white
  px-4 py-2
  font-mono uppercase tracking-wider text-sm
  shadow-[2px_2px_0px_0px_#000000]
  hover:translate-y-[1px] hover:translate-x-[1px] hover:shadow-none
  transition-none
">
  Submit
</button>
```

按钮变体：Primary（蓝）/ Success（绿）/ Destructive（红）/ Warning（橙）/ Outline（透明）。每屏只有一个 Primary 按钮。

### 7.4 文本域 Enter 键（`docs/portable/swiss-design-system/components.md:78-85`）

模态框或拖拽卡片中嵌入的 textarea 必须阻止 Enter 冒泡：

```tsx
const handleKeyDown = (e: React.KeyboardEvent<HTMLTextAreaElement>) => {
  if (e.key === 'Enter') e.stopPropagation();
};
```

## 8. 端到端用户流程图

下面是从打开站点到导出 PDF 的完整路径（结合 `apps/frontend/CLAUDE.md:14-32`）：

```
[浏览器] 打开 http://localhost:3000
   │
   ▼
[/]                (app/(default)/page.tsx, server)  ←  Landing
   │  Hero + 介绍
   ▼
用户点击 "Get Started"
   │
   ▼
[/dashboard]       (client)  ←  Resume 列表
   │  - 状态网格：master / tailored / failed
   │  - 操作：Upload、Delete、Rename、Retry
   │
   ▼
[/builder]         (client → components/builder/resume-builder.tsx)
   │  1) 主简历编辑（个人 / 摘要 / 经历 / 项目 / 教育 / 技能 ...）
   │  2) 模板选择 + 格式化控件（margin / font / accent）
   │  3) AI Regenerate 单条目 / 单章节
   │  4) 拖拽排序、自定义章节、增删可见性
   │
   ├──► AI Enrichment 向导（点击 "Enhance Resume"）
   │       analyze → 6 道题 → enhance → apply
   │
   ▼
[/tailor]          (client)  ←  粘贴 JD
   │ 1) POST /resumes/improve/preview
   │ 2) 服务端执行：extract_job_keywords → skill_target_plan → diff → verify
   │ 3) 返回 diff，用户在 diff modal 中确认
   │ 4) POST /resumes/improve/confirm（携带 preview_hash）
   │
   ▼
[/resumes/[id]]    (client)  ←  单份简历查看
   │  - JD MATCH 标签（仅 tailored 简历出现）
   │     - 左：原始 JD；右：关键词高亮简历
   │  - Download PDF 按钮
   │
   ▼
[后端] GET /resumes/{id}/pdf
   │ 1) 构造 URL：{FRONTEND_BASE_URL}/print/resumes/{id}?template=...
   │ 2) Playwright 启动 headless Chromium
   │ 3) 等待 .resume-print + document.fonts.ready
   │ 4) page.pdf(format=A4, print_background=True, margins=...)
   │ 5) 返回 PDF 字节流
   │
   ▼
[浏览器] 文件下载
   │
   ▼
（可选）打开 /print/cover-letter/[id]，再次走 Playwright 导出求职信 PDF
```

### 关键交互点

- **PDF 路由是 Server Component**（`app/print/resumes/[id]/page.tsx`），从 `searchParams` 读取模板参数，通过 `API_BASE` 服务端直接调用后端获取数据。不要给 `print/*` 加 `'use client'`。
- **Margin 实际由 Playwright 应用**（`docs/agent/design/print-pdf-design-spec.md:35-39`），HTML 里置零边距以保证多页一致。
- **Page Break** 由 `apps/frontend/components/preview/use-pagination.ts` 计算，确保 `.resume-item` 不跨页、章节标题不孤立。
- **健康检查**仅访问内部后端 `127.0.0.1:8000/api/v1/health`（见 `Dockerfile:123-124`）。
